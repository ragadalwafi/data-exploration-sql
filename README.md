# Sales Data Exploratory Analysis (SQL)
---
## Project Overview
This project focuses on performing Exploratory Data Analysis (EDA) on a sales data warehouse using SQL.
The goal of this analysis is to explore the database structure, understand customer and product behavior, and generate key business insights from sales transactions.

---
## Dataset Structure
The project uses a data warehouse schema containing the following tables:

- gold.dim_customers → Contains customer information such as name, gender, country, and birthdate.

- gold.dim_products → Contains product details including category, subcategory, and cost.

- gold.fact_sales → Contains transactional sales data such as orders, quantities, prices, and revenue.

---
## Objectives

The main objectives of this project are:

- Explore database tables and columns

- Analyze customer distribution by country and gender

- Explore product hierarchy (Category → Subcategory → Product)

- Calculate key business metrics such as total sales and total orders

- Identify top performing products and customers

- Analyze revenue distribution across different categories and regions

  ## Skills Demonstrated

## Example Insights

Some insights that can be generated from this analysis include:

- Which product categories generate the highest revenue

- Which customers contribute the most sales

- Sales distribution across countries

- Product performance within each category

## Project Purpose

This project demonstrates how SQL can be used for real-world data exploration and business intelligence analysis. It highlights the importance of understanding data before building dashboards or predictive models.
